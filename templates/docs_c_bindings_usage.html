C bindings source preview

Use Lamina from C

The development crate already exposes a stable-shaped AOT API: create owned handles, build or provide Lamina IR, emit assembly, then release every returned resource explicitly.

Source-preview ABI

The examples use the current hand-maintained lamina.h. Review the header from your checkout before integrating it: symbol names and typedefs may still change before the first archive is published.

Ownership rules

BuilderFree with lia_builder_free(). A finished module remains valid after the builder is released.

Types and valuesFree temporary handles after the builder call that consumes them. The builder clones their data.

ModuleFree with lia_module_free() after IR inspection or assembly generation is complete.

BufferRead exactly len bytes, then call lia_buffer_free(). Returned buffers are not C strings.

Nightly JITFunction pointers remain valid only while their owning JIT handle is alive.

Build and inspect an IR module

This source-preview example creates @main, computes 40 + 2, serializes the module, and releases every handle.

#include <stdio.h>
#include <stdlib.h>
#include <lamina.h>

static void require_ok(lia_status_t status) {
    if (status == LIA_OK) {
        return;
    }
    fputs("lamina-c call failed\n", stderr);
    exit(1);
}

int main(void) {
    lia_builder_t *builder = lia_builder_create();
    lia_type_t *i64 = lia_type_i64();
    lia_value_t *forty = lia_value_i64(40);
    lia_value_t *two = lia_value_i64(2);

    require_ok(lia_builder_function(builder, "main", NULL, 0, i64));
    require_ok(lia_builder_binary(
        builder, LIA_BIN_ADD, "answer", i64, forty, two
    ));

    lia_value_t *answer = lia_value_var("answer");
    require_ok(lia_builder_return(builder, i64, answer));

    lia_module_t *module = NULL;
    require_ok(lia_builder_finish(builder, &module));

    lia_buffer_t ir = {0};
    require_ok(lia_module_emit_ir(module, &ir));
    fwrite(ir.data, 1, ir.len, stdout);

    lia_buffer_free(&ir);
    lia_module_free(module);
    lia_value_free(answer);
    lia_value_free(two);
    lia_value_free(forty);
    lia_type_free(i64);
    lia_builder_free(builder);
    return 0;
}

Compile a module to assembly

Use a null target to compile for the host, or pass an explicit tested target such as x86_64_linux or x86_64_windows. The current source preview accepts only opt_level = 0.

lia_compile_options_t options = {
    .target = "x86_64_linux",
    .codegen_units = 1,
    .opt_level = 0,
};

lia_buffer_t assembly = {0};
require_ok(lia_module_compile_to_assembly(module, &options, &assembly));
fwrite(assembly.data, 1, assembly.len, stdout);
lia_buffer_free(&assembly);
Assembly output only

The current stable-shaped C API emits assembly bytes. Object-file generation, executable linking helpers, and packaged compiler-driver integration remain follow-up work.

Nightly JIT extension

Runtime compilation is an optional nightly API. The source crate gates these symbols behind --features nightly: lia_jit_compile_ir, lia_module_compile_jit, lia_jit_get_function, lia_jit_call_i64_0, lia_jit_call_i64_1, lia_jit_call_i64_2, and lia_jit_free.

No stable JIT ABI

These declarations live in lamina_nightly.h, not lamina.h. Nightly users should expect changes.
