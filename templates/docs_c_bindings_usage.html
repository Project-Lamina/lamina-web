C bindings source preview

Use Lamina from C

The development crate already exposes a stable-shaped AOT API: create owned handles, build or provide Lamina IR, emit assembly, then release every returned resource explicitly.

Header is still pending

The examples use the intended public typedef aliases for the forthcoming lamina.h. Exported function names match the current lamina-c source, but typedef spelling may still change before the first archive is published.

Ownership rules

BuilderFree with lamina_builder_free(). A finished module remains valid after the builder is released.

Types and valuesFree temporary handles after the builder call that consumes them. The builder clones their data.

ModuleFree with lamina_module_free() after IR inspection or assembly generation is complete.

BufferRead exactly len bytes, then call lamina_buffer_free(). Returned buffers are not C strings.

Nightly JITFunction pointers remain valid only while their owning JIT handle is alive.

Build and inspect an IR module

This source-preview example creates @main, computes 40 + 2, serializes the module, and releases every handle.

#include <stdio.h>
#include <stdlib.h>
#include <lamina.h>

static void require_ok(lamina_status_t status) {
    if (status == LAMINA_OK) {
        return;
    }
    fputs("lamina-c call failed\n", stderr);
    exit(1);
}

int main(void) {
    lamina_builder_t *builder = lamina_builder_create();
    lamina_type_t *i64 = lamina_type_i64();
    lamina_value_t *forty = lamina_value_i64(40);
    lamina_value_t *two = lamina_value_i64(2);

    require_ok(lamina_builder_function(builder, "main", NULL, 0, i64));
    require_ok(lamina_builder_binary(
        builder, LAMINA_BINARY_ADD, "answer", i64, forty, two
    ));

    lamina_value_t *answer = lamina_value_var("answer");
    require_ok(lamina_builder_return(builder, i64, answer));

    lamina_module_t *module = NULL;
    require_ok(lamina_builder_finish(builder, &module));

    lamina_buffer_t ir = {0};
    require_ok(lamina_module_emit_ir(module, &ir));
    fwrite(ir.data, 1, ir.len, stdout);

    lamina_buffer_free(&ir);
    lamina_module_free(module);
    lamina_value_free(answer);
    lamina_value_free(two);
    lamina_value_free(forty);
    lamina_type_free(i64);
    lamina_builder_free(builder);
    return 0;
}

Compile a module to assembly

Use a null target to compile for the host, or pass an explicit tested target such as x86_64_linux or x86_64_windows. The current source preview accepts only opt_level = 0.

lamina_compile_options_t options = {
    .target = "x86_64_linux",
    .codegen_units = 1,
    .opt_level = 0,
};

lamina_buffer_t assembly = {0};
require_ok(lamina_module_compile_to_assembly(module, &options, &assembly));
fwrite(assembly.data, 1, assembly.len, stdout);
lamina_buffer_free(&assembly);
Assembly output only

The current stable-shaped C API emits assembly bytes. Object-file generation, executable linking helpers, and packaged compiler-driver integration remain follow-up work.

Nightly JIT extension

Runtime compilation is an optional nightly API. The source crate gates these symbols behind --features nightly: lamina_jit_compile_ir, lamina_module_compile_jit, lamina_jit_get_function, lamina_jit_call_i64_0, lamina_jit_call_i64_1, lamina_jit_call_i64_2, and lamina_jit_free.

No stable JIT ABI

The published SDK will place these declarations in lamina_nightly.h, not lamina.h. Nightly users should expect changes.
