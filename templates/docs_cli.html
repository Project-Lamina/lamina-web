Compiler driver

The lamina binary builds executables, runs code through the JIT, emits MIR or assembly, and selects targets and tools.

$ lamina <input.lamina> [options]
OptionArgumentBehavior

-o, --output<file>Choose the output artifact name.

-v, --verbose-Print additional compiler-driver output.

--target<arch_os>Select a target such as x86_64_linux.

--opt-level<0-3>Choose the experimental MIR optimization level. Default: 1.

--emit-mir-Write .lumir MIR and exit.

--emit-mir-asm-Experimental: emit assembly from MIR, using --target for architecture and OS.

--emit-asm-Emit an assembly file without assembling or linking it.

--jit-Compile and execute through the runtime compiler.

--sandbox-Enable sandboxed JIT execution. Requires --jit.

-c, --compiler<name>Select gnu, clang, msvc, lamina, or a linker-only backend.

--assembler<name>Select ras, gas, or clang.

-Wl,<flag>-Pass comma-separated flags to the linker.

-Wa,<flag>-Pass comma-separated flags to the assembler.

-j, --jobs<n>Set parallel code-generation threads. Default: available cores minus two.

--regalloc<method>Select incremental, linear, or graph.

-g-Emit assembler debug lines and minimal ELF DWARF with ras.

-h, --help-Print the built-in command reference.

--version-Print the compiler version.

Toolchain selection

The --compiler option accepts complete toolchains and linker-only choices. Complete choices are gnu, clang, msvc, and lamina. Linker-only choices are ld, lld, mold, weld, link, and cl.

$ lamina app.lamina --compiler lamina --assembler ras
$ lamina app.lamina --compiler clang -Wl,-dead_strip
$ lamina app.lamina --emit-mir --opt-level 3 --regalloc graph
