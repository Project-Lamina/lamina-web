Quick start

Compile Lamina IR from the command line.

Lamina accepts a compact, typed SSA representation. Install the crate, write an IR module, and compile it into a native executable. 

$ cargo install lamina
$ lamina app.lamina --opt-level 2
$ ./app

Use --target for cross-target emission and --emit-asm to stop before assembly and linking.

Pipeline

Keep every compiler stage visible.

Lamina moves through a small set of explicit stages:

  1. Parse IRRead typed SSA functions, blocks, globals, values, and memory operations.
  2. Lower to MIRConvert architecture-neutral IR into a machine-oriented representation.
  3. TransformRun optimization passes selected by the configured optimization level.
  4. Generate codeSelect target instructions, allocate registers, and apply the selected ABI.
  5. DeliverEmit assembly, link an AOT executable, or compile into executable memory for JIT mode.

AOT compilation

Build native artifacts ahead of time.

AOT mode is the default CLI path. It emits target code and produces a native executable for delivery. You can stop earlier when inspecting compiler output. 

$ lamina app.lamina --opt-level 2
$ lamina app.lamina --emit-mir
$ lamina app.lamina --emit-asm
$ lamina app.lamina --target aarch64_macos

JIT mode

Compile and execute at runtime.

JIT mode lowers Lamina IR into executable memory on the host. Use the CLI for direct execution or embed the runtime compiler from Rust. 

$ lamina app.lamina --jit
$ lamina app.lamina --jit --sandbox

The runtime surface also exposes the inline lamina! macro and JIT helpers for Rust callers.

IRBuilder

Generate IR without formatting strings.

The Rust IRBuilder API constructs modules, functions, blocks, typed operations, control flow, memory operations, and returns programmatically. 

use lamina::ir::{BinaryOp, IRBuilder, PrimitiveType, Type};
use lamina::ir::builder::{i32, var};

let mut builder = IRBuilder::new();
builder
    .function("add", Type::Primitive(PrimitiveType::I32))
    .binary(BinaryOp::Add, "sum", PrimitiveType::I32, i32(20), i32(22))
    .ret(Type::Primitive(PrimitiveType::I32), var("sum"));

The C bindings roadmap includes an owned C wrapper for this builder surface.

C bindings

A stable C API is planned as a supported integration layer.

The C API will expose opaque handles with explicit ownership, copied strings, thread-local errors, and panic containment. It will cover:

  • IRBuilder functions for values, types, functions, blocks, memory, control flow, and typed operations.
  • AOT compilation from textual IR or built modules.
  • JIT compilation with executable-memory ownership retained by the JIT handle.
  • Generated lamina.h, C examples, and CI smoke tests.
Status

The stable C header is being designed. Do not depend on internal Rust ABI details.
