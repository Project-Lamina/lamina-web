Rust API guide

Use the library helpers to compile IR from Rust. For exact types and item signatures, open the generated docs on docs.rs.

Published reference

The generated API reference for Lamina 0.0.8 is on docs.rs. This guide covers common tasks; docs.rs lists every public Rust item.

Compile IR to host assembly

use lamina::compile_lamina_ir_to_assembly;

let ir = r#"
fn @main() -> i64 {
  entry:
    %answer = add.i64 40, 2
    ret.i64 %answer
}
"#;

let mut assembly = Vec::new();
compile_lamina_ir_to_assembly(ir, &mut assembly, 1)?;

Compile for a target

Use compile_lamina_ir_to_target_assembly when your frontend needs assembly for a specific architecture_os target.

use lamina::compile_lamina_ir_to_target_assembly;

let mut assembly = Vec::new();
compile_lamina_ir_to_target_assembly(
    ir,
    &mut assembly,
    "x86_64_linux",
    1,
)?;
Assembly helpers

Both helper functions parse IR text and write assembly into any std::io::Write destination. The host helper detects the current target. The target helper validates an explicit architecture_os string.

Nightly runtime compilation

Runtime compilation is not part of the normal release surface. Enable the nightly feature before using the inline macro or lower-level JIT APIs.

$ cargo add lamina --features nightly
use lamina::lamina;

let add = lamina!(r#"
fn @add(i64 %left, i64 %right) -> i64 {
  entry:
    %sum = add.i64 %left, %right
    ret.i64 %sum
}
"#);

assert_eq!(add(&[20, 22]), Some(42));
Current macro limits

The inline macro is a nightly API, compiles at runtime, and currently uses i64 parameters and return values.

Own executable memory with nightly

The lower-level runtime API returns executable memory and a function pointer. Keep the memory alive for every call through that pointer. Treat the pointer as unsafe: its Rust or C signature must match the compiled function.
