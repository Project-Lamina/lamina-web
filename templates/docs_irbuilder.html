IRBuilder reference

IRBuilder constructs typed modules without writing IR text by hand. Start a function, append instructions to the active block, then call build().

use lamina::ir::{BinaryOp, IRBuilder, PrimitiveType, Type};
use lamina::ir::builder::{i32, var};

let mut builder = IRBuilder::new();
builder
    .function("add", Type::Primitive(PrimitiveType::I32))
    .binary(BinaryOp::Add, "sum", PrimitiveType::I32, i32(20), i32(22))
    .ret(Type::Primitive(PrimitiveType::I32), var("sum"));

let module = builder.build();
Borrowed names

IRBuilder<'a> stores borrowed names and string values. Keep the backing strings alive until the module is finished. The C API uses an owned builder wrapper so C callers do not need to manage Rust lifetimes.

Do not ship print

IRBuilder::print() is intended for compiler debugging and is not available in normal Lamina releases. Use write_byte() or the raw I/O methods for program output.

Functions and blocks

function() creates an entry block and selects it as the active block. Call block() to add another block and make it active. Use set_entry_block() only when a different block should run first.

Annotations

Function helpers include inline, export, external, no_return, hot, cold, pure, const_fn, internal, visibility controls, alignment, section selection, and calling-convention helpers such as cc_c.
