Targets and backends

Linux and Windows x86_64 are tested end to end. Apple Silicon macOS has received partial testing. The remaining published paths are experimental.

These paths have code-generation support in the source tree, but they are not part of the tested set yet. Expect incomplete operations, toolchain limitations, or linker issues.

How to read this list

Target parsing, code generation, assembly, and linking are separate stages. A recognized identifier does not promise an end-to-end compiler path.

Driver acceptanceThe command-line parser recognizes more identifiers than this public matrix. Recognition alone is not a support claim.

AArch64 binary emissionThe MIR backend still reports direct binary emission as unimplemented. Assembly-based workflows are separate from that limitation.

Experimental register allocationGlobal linear-scan and graph-color allocation are not available for every backend. The default allocator remains the practical choice.
