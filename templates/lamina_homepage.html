Typed SSA compiler backend

Follow the compiler from IR to machine code.

Lamina compiles a compact, typed SSA IR. Use it to build native executables, compile functions at runtime, or give a new language frontend a backend.

Compiler pipeline

The driver can stop after MIR or assembly, which makes each stage easier to inspect while you work.

01

program.lamina

Parse Lamina IR

Read typed, block-based SSA with functions, globals, pointers, and explicit memory operations.

02

module.lumir

Lower to MIR

Lower target-independent operations into the machine-oriented IR used by later passes.

03

-O0 ... -O3

Run transforms

Apply CFG simplification, constant folding, dead-code removal, copy propagation, inlining, and strength reduction.

04

target.s / target.wat

Generate target code

Select instructions, allocate registers, build stack frames, and apply the selected target ABI.

05

native executable

Assemble and link

Create a native executable in AOT mode. When debugging a build, stop after MIR or assembly.

Execution modes

Build an executable or compile at runtime.

AOT and JIT modes use the same IR and lowering path.

01 / AOT

Ahead-of-time builds

Compile Lamina IR into target assembly and link a native executable. The driver lets you choose the target, optimization level, and register allocator.

lamina app.lamina

02 / JIT

Runtime compilation

Lower IR into executable memory and run it on the host. Use the CLI, the Rust API, or the inline lamina! macro.

lamina app.lamina --jit

Compiler driver

Run the full build or stop after one stage.

The common case stays short. For debugging, the CLI can emit MIR or assembly, change targets, tune optimization, and choose a register allocator.
