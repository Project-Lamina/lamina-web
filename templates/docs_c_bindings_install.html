C bindings development SDK

Install the C bindings

The rolling development release publishes Linux x86_64 SDK archives for the AOT API and the optional nightly JIT extension. Each archive contains headers, static and shared libraries, examples, pkg-config metadata, and a license.

Development ABI

The archives are ready for evaluation and integration testing. The C ABI may still change before the first stable SDK release.

Current progress

Stable cratelamina-c exists and builds as a static library and shared library.

AOT APIOwned builder handles, module serialization, assembly compilation, buffers, and status codes are implemented.

Nightly JITThe optional JIT module builds behind --features nightly.

Public headerslamina.h and lamina_nightly.h exist. They are maintained by hand until cbindgen supports Rust 2024 #[unsafe(no_mangle)].

Example checksThe stable AOT, raw IR, and nightly JIT examples build and run against the published Linux archives.

Release bundlesAOT and nightly JIT Linux x86_64 tarballs are published on the rolling GitHub prerelease.

Download the Linux SDK

Choose the AOT archive for non-JIT integration. Download the nightly archive only when you need the experimental JIT API.

ChannelArchiveUse it for

AOTlamina-c-0.1.0-stable-x86_64-linux.tar.gzlamina.h, shared and static libraries, AOT builder example, and raw IR example.

Nightly JITlamina-c-0.1.0-nightly-x86_64-linux.tar.gzStable files plus lamina_nightly.h and the JIT example.

$ curl -LO https://github.com/SkuldNorniern/lamina/releases/download/latest/lamina-c-0.1.0-stable-x86_64-linux.tar.gz
$ tar -xzf lamina-c-0.1.0-stable-x86_64-linux.tar.gz
$ cd lamina-c-0.1.0-stable-x86_64-linux
$ cc examples/builder_aot.c -I include -L lib -llamina_c \
    -Wl,-rpath,'$ORIGIN/lib' -o builder_aot
$ ./builder_aot
Checksum note

The current .sha256 companion files contain correct digests but include CI-runner paths, so sha256sum -c is not portable yet. This is tracked as a packaging fix for the next rolling build.

Build from a source checkout

A source build is useful when testing current development changes. Build the standalone lamina-c crate through its manifest.

$ git clone https://github.com/SkuldNorniern/lamina.git
$ cd lamina
$ cargo build --manifest-path lamina-c/Cargo.toml --release

Apple Silicon macOS

The aarch64_macos compiler path has received partial testing. Treat macOS C binding builds as preview work until release coverage is broader.

Build the nightly JIT extension

JIT is not part of the stable C surface. Build with the nightly feature only when testing runtime compilation.

$ cargo build --manifest-path lamina-c/Cargo.toml --release --features nightly

JIT declarations live in the additive lamina_nightly.h header. Stable programs should include only lamina.h.
