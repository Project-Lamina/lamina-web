IR language reference

Lamina IR is readable text built from blocks and typed SSA values. Function names start with @; local values start with %.

Module shape

fn @select_larger(i64 %left, i64 %right) -> i64 {
  entry:
    %is_larger = gt.i64 %left, %right
    br %is_larger, left_wins, right_wins
  left_wins:
    ret.i64 %left
  right_wins:
    ret.i64 %right
}

Memory ownership

alloc.stack creates function-scoped storage with automatic cleanup. alloc.heap creates persistent storage that must be released with dealloc.

fn @round_trip(i64 %input) -> i64 {
  entry:
    %slot = alloc.stack i64
    store.i64 %slot, %input
    %loaded = load.i64 %slot
    ret.i64 %loaded
}

Control-flow rules

A function contains named basic blocks. Execution starts at its entry block, and each path ends with a terminator such as ret, br, jmp, or switch. Use phi when a block needs one SSA value selected from multiple incoming edges.

Portable output

Use writebyte for portable text output.

Debug-only helper

print is for compiler debugging. It is not available in normal Lamina releases and must not be used for application output. Use writebyte or the raw I/O instructions instead.
