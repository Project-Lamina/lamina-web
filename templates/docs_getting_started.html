Getting started

Install the driver and compile a small typed IR module. Then choose AOT, JIT, or an inspection output.

What Lamina is

A frontend can emit Lamina IR text or build a module with IRBuilder. Lamina lowers the module, runs optimization passes, and generates target code.

Install the driver

$ cargo install lamina
$ lamina --help

When working from a source checkout, replace lamina with cargo run --.

Compile a first module

fn @main() -> i64 {
  entry:
    %answer = add.i64 40, 2
    ret.i64 %answer
}
$ lamina answer.lamina
$ ./answer
$ echo $?
42

The default path is AOT compilation: Lamina emits target assembly, assembles it, and links an executable. The process exit status is 42 because @main returns the computed value.

Choose an output mode

AOTThe default path. Generate assembly and link a native executable.

JITUse --jit to compile and execute on the host at runtime.

Inspect MIRUse --emit-mir to write a .lumir file and exit.

Inspect assemblyUse --emit-asm to stop before assembling and linking.

$ lamina answer.lamina
$ lamina answer.lamina --jit
$ lamina answer.lamina --emit-mir --opt-level 2
$ lamina answer.lamina --emit-asm --target aarch64_macos

Work from a source checkout

Use cargo run -- when testing a local checkout. Everything after -- is passed to the Lamina driver.

$ cargo run -- answer.lamina --emit-mir
$ cargo run -- answer.lamina --jit
