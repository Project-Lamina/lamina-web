Development SDK

C bindings progress

lamina-c wraps Lamina's IRBuilder, module serialization, AOT assembly output, diagnostics, and cleanup rules for C programs. A separate nightly header adds the experimental JIT interface.

Status: development archives

Linux x86_64 SDK archives are available from the rolling development release. The ABI is still under review, so use the current headers as a development interface rather than a long-term compatibility promise.

Implemented now

Owned builderOwnedIRBuilder removes Rust lifetime requirements from the FFI boundary.

AOT surfaceRaw IR text or a built module can be compiled into an assembly byte buffer.

Explicit cleanupBuilders, modules, temporary types, values, buffers, and nightly JIT handles have matching free functions.

C headerslamina.h defines the AOT surface. The additive lamina_nightly.h header contains the experimental JIT API. Both are maintained by hand while cbindgen lacks Rust 2024 support.

Nightly JITRuntime compilation and typed i64 call helpers are isolated behind the nightly feature.

Linux archivesThe rolling development release includes stable and nightly Linux x86_64 tarballs with headers, libraries, examples, metadata, and a license.

Example checksThe stable AOT, raw IR, and nightly JIT examples build and run against the published Linux archives.

Before a stable SDK release

  • Header automationTrack cbindgen support for Rust 2024 #[unsafe(no_mangle)]. Until then, review the hand-maintained headers with each ABI change.
  • Clean CI rebuildRebuild the C examples from scratch in CI so the lia_* exports, headers, and examples cannot drift apart.
  • ChecksumsGenerate portable checksum files. The current rolling release records the correct digest but writes a CI-runner path into each .sha256 file.
  • More platformsAdd packaged smoke tests before publishing Windows or Apple Silicon macOS archives.
  • ABI reviewFinish naming, ownership, and error-contract review before promising compatibility across releases.
