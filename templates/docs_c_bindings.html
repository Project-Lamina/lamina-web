C bindings progress

The development checkout now contains a real lamina-c crate. It wraps owned IRBuilder state, module serialization, AOT assembly output, explicit resource cleanup, and an optional nightly JIT module.

Status: source preview

The crate builds and includes hand-maintained stable and nightly headers, but it is not a published SDK yet. Packaged smoke tests, release archives, checksums, and the final ABI review remain open.

Implemented now

Owned builderOwnedIRBuilder removes Rust lifetime requirements from the FFI boundary.

Stable-shaped AOTRaw IR text or a built module can be compiled into an assembly byte buffer.

Explicit cleanupBuilders, modules, temporary types, values, buffers, and nightly JIT handles have matching free functions.

C headerslamina.h and additive lamina_nightly.h headers exist in the development checkout. They are maintained by hand while cbindgen lacks Rust 2024 support.

Nightly JITRuntime compilation and typed i64 call helpers are isolated behind the nightly feature.

Example checksmake test and make test-nightly build and run the stable AOT, raw IR, and nightly JIT examples.

Before the first public archive

  • Header automationTrack cbindgen support for Rust 2024 #[unsafe(no_mangle)]. Until then, review the hand-maintained headers with each ABI change.
  • Clean CI rebuildRebuild the C examples from scratch in CI so the lia_* exports, headers, and examples cannot drift apart.
  • TestsCompile C smoke programs against packaged Linux and Windows archives.
  • DistributionPublish versioned GitHub Release archives and SHA-256 checksums, then expose them through this website.
